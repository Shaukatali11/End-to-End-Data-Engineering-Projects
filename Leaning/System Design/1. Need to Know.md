# Cross Questions

## Q1

Why Kafka?

Answer

Kafka supports high-throughput, fault-tolerant, scalable event streaming. It decouples producers (applications) from consumers (analytics, fraud detection, notifications), allows replay of events, and can handle millions of events per second.

---

## Q2

Why Medallion Architecture?

Answer

It separates concerns:

* Bronze preserves raw history.
* Silver applies cleaning and validation.
* Gold provides business-ready datasets.

This improves maintainability, reprocessing, and trust in analytical data.

---

## Q3

How do you handle duplicate orders?

Answer

Use a unique business key such as `order_id`. During the Silver transformation, perform deduplication using the latest event timestamp or version number. Maintain idempotent processing so re-running the pipeline does not create duplicates.

---

## Q4

What if Kafka goes down?

Answer

Deploy Kafka in a multi-broker cluster with replication. Producers should use acknowledgments (`acks=all`), and consumers should commit offsets only after successful processing. Alerts and retry mechanisms should be configured.

---

## Q5

How would you process late-arriving data?

Answer

Use event time rather than processing time. Implement watermarking for streaming jobs where appropriate, and periodically reprocess affected partitions in batch to ensure consistency.

---

## Q6

How would you design for 100 million orders per day?

Answer

* Partition data by date and region.
* Use distributed storage such as object storage or a data lake.
* Scale Kafka partitions horizontally.
* Use distributed processing engines like Spark.
* Store analytical data in a scalable cloud warehouse.
* Optimize files using compaction and appropriate file sizes.

---

## Q7

Why not directly load data into the Data Warehouse?

Answer

Raw data often contains errors, duplicates, schema changes, and incomplete records. Loading directly into the warehouse makes analytics unreliable. The data lake and Medallion architecture provide a controlled staging and transformation process.

---

## Q8

How do you ensure data quality?

Answer

Implement automated validation checks such as:

* Null checks
* Duplicate detection
* Referential integrity
* Accepted value validation
* Range checks
* Record count reconciliation
* Schema validation

Failed records should be quarantined for investigation instead of silently discarded.

---

## Q9

How do you make pipelines fault tolerant?

Answer

* Design idempotent jobs.
* Enable checkpointing for streaming.
* Implement retries with exponential backoff.
* Store metadata and processing status.
* Monitor failures and trigger alerts.
* Allow pipelines to resume from the last successful checkpoint.
