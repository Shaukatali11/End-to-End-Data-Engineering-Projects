## Design a Data Platform for an E-commerce Company

---

# Interviewer

Imagine you have joined Amazon as a Senior Data Engineer.

You need to design the complete Data Platform for Amazon.

The platform should support:

* Product Search
* Product Recommendation
* Inventory
* Payments
* Orders
* Delivery
* Analytics
* Machine Learning
* Real-time Dashboards

How would you design it?

---

# Candidate

Before designing anything, I never jump into architecture.

I first gather requirements because architecture depends on business needs.

I divide the discussion into four parts:

1. Functional Requirements
2. Non-Functional Requirements
3. Scale Estimation (Back-of-the-Envelope)
4. High-Level Architecture

# Microservices

Instead of one monolithic producer, separate services publish events independently:

* User Service → User events
* Product Service → Catalog updates
* Cart Service → Cart events
* Order Service → Order lifecycle
* Payment Service → Payment status
* Inventory Service → Stock changes
* Shipping Service → Delivery updates
* Notification Service → Email/SMS events

Each service owns its data and publishes to dedicated Kafka topics. This decouples systems so failures in one service do not stop the others.
