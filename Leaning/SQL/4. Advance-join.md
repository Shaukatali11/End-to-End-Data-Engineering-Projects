# 🔗 What is JOIN?

A **JOIN** combines records from two or more tables using a common column.

### Why do we use JOIN?

- Normalize database design
- Reduce data redundancy
- Fetch related data
- Generate reports
- Build ETL pipelines
- Create analytical datasets

### JOIN Syntax

```sql
SELECT columns
FROM Table1
JOIN Table2
ON Table1.column = Table2.column;
```

# 1️⃣ INNER JOIN

## Definition

Returns **only matching records** from both tables.

If there is no match, the row is ignored.

### Syntax

```sql
SELECT *
FROM Employee e
INNER JOIN Department d
ON e.Dept_ID=d.Dept_ID;
```

### Example

```sql
SELECT
e.Name,
d.Department
FROM Employee e
INNER JOIN Department d
ON e.Dept_ID=d.Dept_ID;
```

### Output

| Name | Department |
|------|------------|
|Shaukat|IT|
|Rahul|HR|
|Aman|IT|

✅ Employee "Neha" is not returned because Dept_ID is NULL.

### Real-world Use Case

- Customer & Orders
- Employee & Department
- Student & Course

---

# 2️⃣ LEFT JOIN

## Definition

Returns **all rows from the Left table** and matching rows from the Right table.

If no match exists, NULL is returned.

### Syntax

```sql
SELECT *
FROM Employee e
LEFT JOIN Department d
ON e.Dept_ID=d.Dept_ID;
```

### Example

```sql
SELECT
e.Name,
d.Department
FROM Employee e
LEFT JOIN Department d
ON e.Dept_ID=d.Dept_ID;
```

### Output

| Name | Department |
|------|------------|
|Shaukat|IT|
|Rahul|HR|
|Aman|IT|
|Neha|NULL|

### Use Cases

- Show all customers even if they never ordered.
- Show all products even if never sold.

---

# 3️⃣ RIGHT JOIN

## Definition

Returns all rows from the Right table and matching rows from the Left table.

If there is no match on the left side, NULL is returned.

### Syntax

```sql
SELECT *
FROM Employee e
RIGHT JOIN Department d
ON e.Dept_ID=d.Dept_ID;
```

### Output

| Department | Name |
|------------|------|
|IT|Shaukat|
|IT|Aman|
|HR|Rahul|
|Finance|NULL|

### Use Cases

- Show all departments.
- Show all categories.
- Show all stores.

---

# 4️⃣ FULL JOIN

## Definition

Returns every row from both tables.

Matched records are merged, unmatched records contain NULL values.

### Syntax

```sql
SELECT *
FROM Employee e
FULL OUTER JOIN Department d
ON e.Dept_ID=d.Dept_ID;
```

### Output

| Name | Department |
|------|------------|
|Shaukat|IT|
|Rahul|HR|
|Aman|IT|
|Neha|NULL|
|NULL|Finance|

### Use Cases

- Data comparison
- Data migration
- Data reconciliation

---

# 5️⃣ CROSS JOIN

## Definition

Returns the Cartesian Product.

Every row from Table A combines with every row from Table B.

### Formula

```
Rows = Table1 × Table2
```

4 Employees

×

3 Departments

=

12 Records

### Syntax

```sql
SELECT *
FROM Employee
CROSS JOIN Department;
```

### Example Output

| Employee | Department |
|-----------|------------|
|Shaukat|IT|
|Shaukat|HR|
|Shaukat|Finance|
|Rahul|IT|
|Rahul|HR|
|...|...|

### Use Cases

- Test data generation
- Matrix reports
- Product combinations

# 6️⃣ SELF JOIN

## Definition

A table joins with itself.

Useful when rows are related to other rows in the same table.

### Example Table

| Employee | ManagerID |
|----------|-----------|
|Shaukat|NULL|
|Rahul|101|
|Aman|101|

### Syntax

```sql
SELECT
e.Name Employee,
m.Name Manager
FROM Employee e
LEFT JOIN Employee m
ON e.ManagerID=m.Emp_ID;
```

### Output

| Employee | Manager |
|----------|----------|
|Rahul|Shaukat|
|Aman|Shaukat|

### Use Cases

- Employee → Manager
- Category → Parent Category
- Family Tree

# 🎯 JOIN Comparison

| JOIN | Returns |
|-------|----------|
| INNER | Only Matching Records |
| LEFT | All Left + Matching Right |
| RIGHT | All Right + Matching Left |
| FULL | All Records from Both Tables |
| CROSS | Every Possible Combination |
| SELF | Same Table Joined with Itself |

# 📚 What are Set Operators?

Set Operators combine the results of two or more SELECT statements.

Rules:

- Same number of columns
- Same data types
- Same column order

---

# 7️⃣ UNION

## Definition

Combines results and removes duplicate rows.

### Syntax

```sql
SELECT Name FROM Employee

UNION

SELECT Name FROM Customer;
```

### Example

Employee

```
Shaukat
Rahul
```

Customer

```
Rahul
Neha
```

Output

```
Shaukat
Rahul
Neha
```

Duplicate "Rahul" removed.

---

# 8️⃣ UNION ALL

## Definition

Combines results and keeps duplicates.

### Syntax

```sql
SELECT Name FROM Employee

UNION ALL

SELECT Name FROM Customer;
```

### Output

```
Shaukat
Rahul
Rahul
Neha
```

### Difference

| UNION | UNION ALL |
|--------|-----------|
|Removes duplicates|Keeps duplicates|
|Slower|Faster|

---

# 9️⃣ INTERSECT

## Definition

Returns only common rows present in both queries.

### Syntax

```sql
SELECT Name FROM Employee

INTERSECT

SELECT Name FROM Customer;
```

### Output

```
Rahul
```

### Use Cases

- Common customers
- Common products
- Common employees

---

# 🔟 EXCEPT

(MySQL equivalent uses `NOT EXISTS` or `LEFT JOIN`.)

## Definition

Returns rows from the first query that do not exist in the second query.

### Syntax

```sql
SELECT Name FROM Employee

EXCEPT

SELECT Name FROM Customer;
```

### Output

```
Shaukat
```

### Use Cases

- Customers who didn't order
- Products never sold
- Employees without projects

---

# 📝 JOIN vs UNION

| Feature | JOIN | UNION |
|----------|-------|--------|
|Combines|Columns|Rows|
|Needs Common Key|✅ Yes|❌ No|
|Output|Wider Table|Longer Table|
|Used For|Related Tables|Merge Result Sets|
