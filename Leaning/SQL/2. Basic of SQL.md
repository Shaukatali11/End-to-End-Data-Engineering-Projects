# 🟢 Level 1: SQL Fundamentals (Beginner)

Learn the building blocks of SQL and relational databases. These concepts form the foundation for writing queries, designing databases, and managing data efficiently.

---

## 1. Introduction to Databases
A database is an organized collection of data that allows easy storage, retrieval, and management.

**Example**
```sql
-- Example: Student database
Student(ID, Name, Age)
```

---

## 2. What is SQL?
SQL (Structured Query Language) is used to communicate with relational databases.

**Syntax**
```sql
SELECT * FROM table_name;
```

---

## 3. Types of Databases
Common types include Relational (MySQL, PostgreSQL), NoSQL (MongoDB), Graph, and Time-Series databases.

**Example**
```text
RDBMS → MySQL
NoSQL → MongoDB
```

---

## 4. RDBMS vs NoSQL

| RDBMS | NoSQL |
|--------|--------|
| Tables | Documents/Key-Value |
| Fixed Schema | Flexible Schema |
| SQL | No SQL |

---

## 5. Database Architecture
A database consists of Client → Database Server → Storage.

**Example**
```text
Application → MySQL → Data
```

---

## 6. Tables, Rows & Columns
A table stores data in rows (records) and columns (attributes).

**Example**
```sql
Employee(ID, Name, Salary)
```

---

## 7. Primary Key
A Primary Key uniquely identifies each row and cannot contain NULL values.

**Syntax**
```sql
CREATE TABLE Employee(
    ID INT PRIMARY KEY
);
```

---

## 8. Foreign Key
A Foreign Key links two tables and maintains referential integrity.

**Syntax**
```sql
FOREIGN KEY (DeptID) REFERENCES Department(ID)
```

---

## 9. Candidate Key
A Candidate Key is any column that can uniquely identify a record.

**Example**
```text
EmployeeID
Email
```

---

## 10. Composite Key
A Composite Key is made using two or more columns.

**Syntax**
```sql
PRIMARY KEY (StudentID, CourseID)
```

---

## 11. Constraints
Constraints enforce rules on data like NOT NULL, UNIQUE, PRIMARY KEY, FOREIGN KEY, CHECK, and DEFAULT.

**Example**
```sql
Age INT CHECK (Age >=18)
```

---

## 12. Data Types
Data types define the kind of values stored in a column.

**Examples**
```text
INT
VARCHAR(100)
DATE
FLOAT
BOOLEAN
```

---

## 13. Creating Database

**Syntax**
```sql
CREATE DATABASE CompanyDB;
```

---

## 14. Creating Table

**Syntax**
```sql
CREATE TABLE Employee(
    ID INT,
    Name VARCHAR(50)
);
```

---

## 15. ALTER Table
Used to modify an existing table.

**Syntax**
```sql
ALTER TABLE Employee
ADD Salary INT;
```

---

## 16. DROP Table
Deletes the table permanently.

**Syntax**
```sql
DROP TABLE Employee;
```

---

## 17. TRUNCATE Table
Removes all rows but keeps the table structure.

**Syntax**
```sql
TRUNCATE TABLE Employee;
```

---

## 18. RENAME Table
Changes the table name.

**Syntax**
```sql
ALTER TABLE Employee
RENAME TO Employees;
```

---

## 19. INSERT Statement
Adds new records into a table.

**Syntax**
```sql
INSERT INTO Employee
VALUES (1,'Shaukat');
```

---

## 20. UPDATE Statement
Modifies existing records.

**Syntax**
```sql
UPDATE Employee
SET Salary=50000
WHERE ID=1;
```

---

## 21. DELETE Statement
Deletes selected records.

**Syntax**
```sql
DELETE FROM Employee
WHERE ID=1;
```

---

## 22. SELECT Statement
Retrieves data from a table.

**Syntax**
```sql
SELECT * FROM Employee;
```

---

## 23. DISTINCT
Removes duplicate values from the result.

**Syntax**
```sql
SELECT DISTINCT Department
FROM Employee;
```

---

## 24. WHERE Clause
Filters records based on a condition.

**Syntax**
```sql
SELECT *
FROM Employee
WHERE Salary > 50000;
```

---

## 25. ORDER BY
Sorts the result in ascending or descending order.

**Syntax**
```sql
SELECT *
FROM Employee
ORDER BY Salary DESC;
```

---

## 26. LIMIT / TOP
Limits the number of returned rows.

**MySQL**
```sql
SELECT *
FROM Employee
LIMIT 5;
```

**SQL Server**
```sql
SELECT TOP 5 *
FROM Employee;
```

---

## 27. Aliases
Provides temporary names to columns or tables.

**Syntax**
```sql
SELECT Name AS Employee_Name
FROM Employee;
```

---

## 28. Comments
Used to explain SQL code.

**Syntax**
```sql
-- Single line comment

/* Multi-line
Comment */
```

---

## 29. NULL Values
NULL represents missing or unknown data.

**Syntax**
```sql
SELECT *
FROM Employee
WHERE Salary IS NULL;
```
